---
rule_id: 1530
rule_category: maintainability
title: Don't change a loop variable inside a `for` loop or a collection in a `foreach` loop
severity: 2
---
Updating the loop variable within the loop body is generally considered confusing, even more so if the loop variable is modified in more than one place.

	for (int index = 0; index < 10; ++index)
	{
		if (someCondition)
		{
			index = 11; // Wrong! Use 'break' or 'continue' instead.
		}
	}

The same applies to the iteration variable in a `foreach` loop. While C# prevents you from reassigning the variable itself, mutating the object it references inside the loop body can cause unexpected behavior and makes the code harder to reason about.

	var names = new List<string> { "Alice", "Bob", "Charlie" };

	foreach (string name in names)
	{
		if (name == "Bob")
		{
			names.Remove(name); // Wrong! Don't modify the collection you're iterating over.
		}
	}
