---
rule_id: 1608
rule_category: testability
title: Show what's important in a test, hide what's not
severity: 2
---
Every test has a specific intent. The data, setup and assertions that are directly relevant to that intent should be visible in the test body. Everything else should be hidden behind well-named helpers or defaults.

```csharp
// Hard to understand: what part actually matters here?
[Fact]
public void Order_is_cancelled_when_payment_fails()
{
    var customer = new Customer("Alice", "alice@example.com", CustomerStatus.Premium, DateTime.UtcNow.AddYears(-2));
    var product = new Product("Widget", 9.95m, ProductCategory.Hardware, "SKU-001", true);
    var order = new Order(customer, new[] { new OrderLine(product, 3) }, "NL-001", DateTimeOffset.UtcNow);
    var gateway = new FakePaymentGateway(succeeds: false);

    order.Pay(gateway);

    order.Status.Should().Be(OrderStatus.Cancelled);
}

// Better: only what's relevant to the test is visible
[Fact]
public void Order_is_cancelled_when_payment_fails()
{
    var order = CreateDefaultOrder();
    var gateway = new FakePaymentGateway(succeeds: false);

    order.Pay(gateway);

    order.Status.Should().Be(OrderStatus.Cancelled);
}
```

Use factory methods or builders to construct objects that are required but irrelevant to the specific assertion.
